# logprompts

현재 세션의 사용자 프롬프트와 Claude 응답 요약을 로그 파일에 저장합니다.

## 로그 위치

`.claude/logs/YYYY-MM-DD.md`

## 규칙

### 사용자 프롬프트
- 사용자가 입력한 프롬프트를 **그대로** 저장
- **문법적인 오류만 수정** (오타, 맞춤법)
- 내용 추가, 삭제, 변경 금지
- 요약하거나 재구성하지 않음

### Claude 응답
- 응답 내용을 **1-3줄로 요약**
- 수행한 작업, 생성/수정한 파일, 주요 결과 포함
- 코드 블록이나 상세 내용은 생략
- 핵심 정보만 간결하게 기록

### 예외: /plan 커맨드
- `/plan` 응답은 **요약하지 않고 전체 내용 그대로 기록**
- 진행률 표, Phase 상태, 완료된 Steps 목록 포함
- 프로젝트 진행 상황 스냅샷으로 활용

## 실행

1. `.claude/logs/` 디렉토리가 없으면 생성
2. 오늘 날짜의 로그 파일 열기 (없으면 생성)
3. 현재 세션의 대화를 시간순으로 기록
4. 타임스탬프와 함께 추가

## 로그 형식

```markdown
## HH:MM

**User**: <사용자 프롬프트 원문 (문법 수정만)>

**Claude**: <응답 요약 1-3줄>
- 수행한 작업 또는 결과
- 생성/수정한 파일 (있는 경우)
```

## 예시

```markdown
## 14:30

**User**: README.md를 영어와 한국어 두 버전으로 만들어 줘.

**Claude**: README 파일을 두 버전으로 분리함.
- README.md: 영어 버전 (한국어 링크 배지 포함)
- README.ko.md: 한국어 버전 신규 생성

---

## 14:35

**User**: /commit

**Claude**: 문서 변경사항 커밋 완료.
- 커밋: `docs: add bilingual README (English + Korean)`
- 변경 파일: README.md, README.ko.md
```
