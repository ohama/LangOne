# plan

현재 계획의 전체 진행 상황을 시각적으로 보여줍니다.

## 사용법

```
/plan              # 전체 계획 상태 표시
/plan summary      # 간단 요약만
/plan phase N      # 특정 Phase 상세 보기
```

## 절차

### 1. 파일 읽기
- .claude/PLAN.md
- .claude/STATE.md

### 2. 진행률 계산
- 전체 Step 수
- 완료된 Step 수
- 현재 진행 중인 Step

### 3. 출력 형식

```
## 📊 LangOne 계획 진행 상황

### 전체 진행률
████████████░░░░░░░░ 35% (6/17 Steps)

### Phase 별 상태

| Phase | 상태 | 진행률 | 진행바 |
|-------|------|--------|--------|
| Phase 0: 프로젝트 초기 설정 | ✅ Completed | 2/2 | ██████████ |
| Phase 1: Lexer (토큰화) | ✅ Completed | 4/4 | ██████████ |
| Phase 2: Parser (구문 분석) | 🔄 In Progress | 0/5 | ░░░░░░░░░░ |
| Phase 3: Evaluator (평가) | ⏳ Not Started | 0/3 | ░░░░░░░░░░ |
| Phase 4: 통합 및 마무리 | ⏳ Not Started | 0/3 | ░░░░░░░░░░ |

### 현재 위치
- **Phase**: Phase 2 - Parser (구문 분석)
- **Step**: Step 2.1 - AST 타입 정의
- **다음**: Step 2.2 - 숫자 리터럴 파싱

### 완료된 Steps (6개)
✓ 0.1 솔루션 및 프로젝트 생성
✓ 0.2 의존성 설치 및 기본 설정
✓ 1.1 기본 타입 정의
✓ 1.2 단일 토큰 Lexing
✓ 1.3 숫자 토큰 Lexing
✓ 1.4 복합 표현식 Lexing
```

### /plan summary

간단한 한 줄 요약:

```
📊 Plan: 35% (6/17) | Phase 2/4 | Current: Step 2.1
```

### /plan phase N

특정 Phase의 상세 정보:

```
## Phase 2: Parser (구문 분석)

**목표**: 토큰 리스트를 AST로 변환하는 Parser 구현

**진행률**: 0/5 Steps (0%)

### Steps

| Step | 상태 | 설명 |
|------|------|------|
| 2.1 | ⏳ Next | AST 타입 정의 |
| 2.2 | ○ | 숫자 리터럴 파싱 |
| 2.3 | ○ | 덧셈/뺄셈 파싱 |
| 2.4 | ○ | 곱셈/나눗셈 파싱 |
| 2.5 | ○ | 괄호 파싱 |

**완료 조건**:
- [ ] 숫자 리터럴 파싱
- [ ] 이항 연산 파싱
- [ ] 연산자 우선순위 처리 (* / > + -)
- [ ] 괄호 처리
- [ ] 에러 처리 (Result 타입)
```

## 진행바 생성 규칙

- 10칸 기준
- 완료: █ (U+2588)
- 미완료: ░ (U+2591)
- 예: 40% = ████░░░░░░

## 상태 아이콘

| 상태 | 아이콘 |
|------|--------|
| Completed | ✅ |
| In Progress | 🔄 |
| Not Started | ⏳ |
| Blocked | 🚫 |

## 규칙

- 코드 변경 없음
- PLAN.md 파싱하여 정보 추출
- 시각적으로 명확하게 표시
- 진행률은 Step 기준으로 계산
